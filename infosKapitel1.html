Erste Schritte mit Processing

Um mit Processing zu programmieren, benötigst du die Processing-Entwicklungsumgebung (IDE). Nach dem Start kannst du direkt Code schreiben und ausführen.

  • Öffne Processing und erstelle ein neues Sketch.
  • Speichere den Sketch direkt auf dem USB-Stick. Überlege dir einen sinnvollen Namen!
  • Schreibe deinen Code in das Editorfenster.
  • Mit einem Klick auf den Play-Button wird dein Programm ausgeführt.

Mit folgendem Beispiel wird ein Kreis in die Mitte des Bildschirms gezeichnet: 

size(100, 100);
circle(50, 50, 40);
Das Koordinatensystem in Processing

Das Koordinatensystem in Processing unterscheidet sich vom mathematischen Koordinatensystem:

  • Der Ursprung (0,0) liegt oben links im Fenster.
  • Die x-Achse verläuft nach rechts
  • Die y-Achse nach unten.

Im mathematischen Koordinatensystem zeigt die y-Achse nach oben. In Processing wächst der y-Wert nach unten.

Die Funktion size(100, 100); legt die Größe des Fensters fest. In diesem Fall ist es 100 Pixel breit und 100 Pixel hoch.

Beispiel: ellipse(100, 50, 50, 50); zeichnet einen Kreis mit Mittelpunkt bei (100, 150).

Kommentare, Anweisungen, Befehlsblöcke
Kommentare

Kommentare kann man zur Beschreibung oder zum Debuggen in sein Programm einfügen. Beim Starten des Programms werden die kommentierten Zeilen ignoriert und nicht ausgeführt.

  • Kommentare sind nützlich, weil der Programmcode viel verständlicher wird. Das gilt für einen selbst sowie andere, die sich das Programm anschauen.
  • Zum anderen kann man Teile seines Codes auskommentieren, um zum Beispiel Fehler zu suchen oder mit neuen Features zu experimentieren.
  • In Processing gibt es einzeilige ("//") und mehrzeilige ( /* ... */ ) Kommentare .

Anweisungen

Ein Computerprogramm besteht aus unterschiedlichen Anweisungen.

  • Funktionsaufrufe (Kapitel 1)
  • Variablendeklarationen und Wertzuweisungen (Kapitel 2)
  • Verzweigungen (if, else) (Kapitel 3)
  • Schleifen (Kapitel 5)
  • und weitere

Hier ein paar Beispiele für Funktionsaufrufe:

Teste es aus! 

print("Hallo Welt!"); //Eine Anweisung, um "Hallo Welt" auf der Konsole auszugeben

ellipse(50, 50, 20, 20); //Eine Anweisung, um einen Kreis auf dem Bildschirm zu zeichnen
Befehlsblöcke
  • Viele Anweisungen zusammen bilden einen Befehlsblock.
  • Wenn Befehlsblöcke in Funktionen, Verzweigungen oder Schleifen stehen, werden sie in Processing von geschweiften Klammern "{ }" umgeben.

Hier ein Beispiele für einen Befehlsblock:

Teste es aus! 

{
  print("Hallo Welt!");
  println("Informatik macht Spaß!");
  println("Wenigstens manchmal ;-)");
}
Syntax, Compiler und Interpreter
Syntax einer Programmiersprache

Die Syntax einer Programmiersprache legt fest, wie Befehle und Anweisungen geschrieben werden müssen, damit sie vom Computer verstanden werden.

  • Jede Programmiersprache hat eigene Regeln für die Schreibweise von Befehlen.
  • Fehler in der Syntax führen dazu, dass das Programm nicht ausgeführt werden kann.
  • Beispiele für Syntaxregeln sind das Setzen von Semikolons, die Verwendung von Klammern oder die korrekte Schreibweise von Schlüsselwörtern.

Beispiel für eine korrekte Syntax in Processing:

circle(100, 100, 50); // Zeichnet einen Kreis mit Mittelpunkt (100, 100) und Radius 50

Wird die Syntax nicht eingehalten, erscheint eine Fehlermeldung.

Beispiel für eine falsche Syntax:

circle(100 100 50) // Fehlende Kommas und Semikolon

Fehlermeldung: SyntaxError: ',' expected

Compiler und Interpreter

Processing/Java ist eine Hochsprache. Das bedeutet, dass die Sprache in einer Form ist, dass sie von Menschen "einfach" geschrieben und verstanden werden kann. 

Der Befehl: circle(100, 100, 100); zeichnet einen Kreis. 

Der Computer, die CPU, kann damit allerdings nichts anfangen. Die Hochsprache muss erst noch durch einen Compiler oder Interpreter in eine für die CPU verständliche Sprache übersetzt werden - den Maschinencode.

Sowohl Interpreter als auch Compiler übersetzen die Hochsprache wie Processing/Java/C++in Maschinencode, der dann von der CPUbearbeitet werden kann.

  • Interpreter - verarbeitet den Programmcode dabei Zeilenweise
  • Compiler - übersetzt den gesamten Code auf einmal
